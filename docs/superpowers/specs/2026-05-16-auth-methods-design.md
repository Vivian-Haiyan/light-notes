# 登录方式扩展设计

## 目标

把登录页从单一邮箱登录扩展为三种入口：

1. 访客体验：使用 Supabase 匿名登录，直接进入应用。
2. 邮箱密码：保留现有正式登录与注册。
3. 手机号验证码：输入手机号后发送验证码，再完成登录。

## 行为约定

- 访客体验会创建匿名用户，拥有独立云端数据空间。
- 匿名用户的侧边栏展示为“访客读者”，邮箱留空展示。
- 手机号登录分两步：发送验证码、校验验证码。
- 手机号登录成功后，如果还没有 `profiles` 记录，则创建默认资料。
- 邮箱注册若开启邮箱确认，则继续提示用户先完成验证。

## 界面结构

- 顶部使用分段切换：`邮箱登录`、`手机号登录`
- 表单下方保留一个独立的 `访客体验` 次级按钮
- 手机号模式中，先输入手机号并点击 `发送验证码`；验证码发送后，再展示验证码输入和 `登录` 按钮
- 邮箱模式继续支持“登录 / 注册”切换

## 错误处理

- 手机号为空、验证码为空时给出表单提示
- 发送验证码失败、验证失败、匿名登录失败都用消息反馈
- 如果 Supabase 项目未启用 Phone Auth 或没有配置短信服务商，保留前端入口，但直接显示来自 Supabase 的错误信息

## 兼容性

- 继续沿用现有 `profiles`、RLS、云端数据仓储
- 访客账号与正式账号都走同一套数据层
- 不引入第三方 OAuth，保持界面克制
